# Seed scripts

One-off bootstrappers used to populate the `production` dataset (and used to
re-seed a fresh dataset, e.g. a `staging` clone). All run from the studio
directory under Node 22.

```bash
cd studio-the-hague-dialogues
export PATH="$HOME/.nvm/versions/node/v22.23.0/bin:$PATH"
```

## Run order (fresh dataset)

1. **`build-seed.mjs`** → writes `seed.ndjson` to this folder.
   ```bash
   node seed/build-seed.mjs
   ```
   Produces ~20 documents (people, topics, venue, partner, events, articles,
   media items, plus the three singletons). All IDs are hyphenated; see
   `~/.claude/projects/.../memory/sanity-setup.md` for the dotted-_id privacy
   gotcha.

2. **`npx sanity dataset import seed/seed.ndjson production --replace`** —
   imports the bundle into the live dataset.

3. **`upload-images.mjs`** — uploads the 4 clean scene photos from
   `public/images/` to Sanity and attaches them to seeded events + media items
   (no new uploads after this; subsequent scripts reuse the resulting asset _ids).
   ```bash
   npx sanity exec seed/upload-images.mjs --with-user-token
   ```

4. **`attach-article-images.mjs`** — reuses uploaded scene assets as article
   covers.
   ```bash
   npx sanity exec seed/attach-article-images.mjs --with-user-token
   ```

5. **`extend-media.mjs`** — adds 2 more `mediaItem` docs so the home media
   mosaic renders 1 feature + 4 stack (matches the prototype layout).
   ```bash
   npx sanity exec seed/extend-media.mjs --with-user-token
   ```

6. **`add-team.mjs`** — uploads the 4 branded team-card images
   (Ádám/Vito/Martin/Viktoria), creates matching `person` docs, and patches
   `aboutContent.team` to reference them.
   ```bash
   npx sanity exec seed/add-team.mjs --with-user-token
   ```

7. **`attach-about-hero.mjs`** — sets `aboutContent.heroImage` to an existing
   uploaded scene-photo asset so the About hero renders via the CMS.
   ```bash
   npx sanity exec seed/attach-about-hero.mjs --with-user-token
   ```

## Notes

- `seed.ndjson` is `.gitignore`d (generated by `build-seed.mjs`).
- All `sanity exec` scripts need `--with-user-token` because they write
  documents and upload assets.
- These scripts are idempotent enough for normal re-runs (`createOrReplace`
  + reused asset _ids) but assume the schema in `schemaTypes/` is already
  deployed.
